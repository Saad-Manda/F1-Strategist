# Commit Command

This command structures Git commits using conventional commit standards.

## Execution Steps

1. Run `git status` and stage files using `git add <file>`.
2. Format commit messages as follows:
   `<type>(<scope>): <short description>`
   - Types: `feat`, `fix`, `docs`, `style`, `refactor`, `test`, `chore`
3. Commit using conventional message.
